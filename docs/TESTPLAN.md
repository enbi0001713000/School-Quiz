# テスト計画（UAT / 受入テスト）

## 目的
公開URL上で、ボタン・出題・採点・履歴が仕様通り動作することを確認する。

## テスト環境
- Chrome（PC）
- iPhone Safari（可能なら）
- 公開URL：GitHub Pages

## チェックリスト

### A. 起動・解除
- [ ] 公開URLを開くとロック解除画面が出る
- [ ] 合言葉が正しいと開始可能になる
- [ ] 合言葉が誤りだと解除されない
- [ ] 名前未入力の場合、解除後に名前入力を促される

### B. 出題
- [ ] 新しいクイズで25問生成される
- [ ] 各教科が5問ずつ含まれる
- [ ] 25問内で同一問題が重複しない
- [ ] 似たpatternが偏りすぎない（極端な連発がない）
- [ ] 問題タグ（教科/学年/難易度/パターン）が表示される

### C. 解答操作
- [ ] 選択肢クリックで選択状態になる（A〜D）
- [ ] 前へ/次へが動作する
- [ ] 解答リセットで全て未回答になる
- [ ] 最終問題で「次へ」が無効になる

### D. 採点・結果
- [ ] 採点で結果カードへ遷移
- [ ] 総合得点が表示される
- [ ] 教科別/難易度別内訳が表示される
- [ ] レーダーチャートが表示される
- [ ] 問題番号クリックで解説が出る
- [ ] 分析文（速度・得意/弱点）が表示される
- [ ] 弱点が無い場合は「偏りがない」旨が出る

### E. 履歴
- [ ] 採点後、履歴が1件増える
- [ ] 履歴を開くボタンで表示される
- [ ] 最近10回グラフが描画される
- [ ] 全期間平均（全体/教科別）が表示される
- [ ] 履歴一覧が最新10件で表示される
- [ ] 履歴全削除が動作する

### F. 更新反映（SW）
- [ ] 更新後に反映されない場合、強制リロードで直る
- [ ] 必要ならswのCACHE_NAME更新で確実に反映される
