# 設計判断ログ（Decisions）

## 方針：社会・理科は固定問題中心
理由：
- 事実関係の誤りが致命傷になりやすい（誤情報＝学習アプリとしての信頼を損なう）
- したがって「固定問題を増やして密度を上げる」戦略を採用

## 方針：数学は思考系テンプレで増量
理由：
- 計算だけだと単調になり、入試難易度の再現性が落ちる
- 関数（変化の割合/交点）、図形（相似/円周角/作図理由）、証明穴埋めを増やす

## 方針：25問内の重複禁止
理由：
- 同じ問題が出ると学習効率が下がり、評価としても不公平
- uid/内容シグネチャで除外する

## 方針：pattern偏り抑制は常時有効
理由：
- 25問が“似た型”だと学習評価が偏る
- UIでの切替は誤操作が増えるため、常時有効にする

## 方針：回答時間は「目安時間」と相対評価
理由：
- 学年/難易度ごとに目安時間を持たせた方が、所見が具体的になる
- 固定秒数ではなく目安比で「速すぎる/遅すぎる」を判断

## 方針：端末内履歴（ログイン不要）
理由：
- 学習の継続を阻害しない（登録不要）
- まずは個人利用の価値を最大化（サーバ同期は将来案）

## 方針：合言葉方式（簡易）
理由：
- URL公開でも「誰でも無制限」は避けたい
- ただし本格的な認証ではない（SECURITY.mdに限界を明記）
