# ポートフォリオ：義務教育5教科クイズ（25問）— 学習×自己分析Webアプリ

## 1. 概要（3行）
義務教育範囲の5教科から25問をランダム出題し、採点・解説・分析（時間傾向含む）を返すWebアプリです。  
端末内に履歴を蓄積し、最近10回の推移と教科別平均から学習方針を立てられます。  
GitHub Pagesで公開し、リンクから即プレイできる形にしました。

## 2. 開発の背景（路線変更を含む）
当初は「遊べるクイズ」を作る目的でした。  
しかし私は大学2年生の頃まで、個別指導塾で講師アルバイトをしており、授業の合間や移動などの隙間時間で
「短時間で学習できる」「弱点が見える」仕組みがあれば学習効率が上がると感じていました。  
そこで路線を変更し、**短時間で受験科目を回しつつ、正誤と時間から自分を分析できる学習アプリ**として開発しました。

## 3. 想定ユーザー
- 中学生（定期テスト・高校入試対策）
- 学び直し（基礎～標準を短時間で確認したい人）
- 学習習慣を作りたい人（短いテスト→分析→次の行動）

## 4. 価値（ユーザーに何が起きるか）
- 25問＝数分で完了し、勉強のハードルが下がる
- 教科別・難易度別の内訳とレーダーで「弱点の可視化」
- 正誤×解答時間の傾向から「ミスのタイプ」を言語化（見落とし型／方針迷い型）
- 端末内履歴により、継続したときの伸びが見える（最近10回推移）

## 5. 主な機能
- 5教科×各5問＝25問、順番ランダム
- 25問内の重複禁止（同じ問題の再出題を抑止）
- 似た出題パターン連発の抑制（patternタグに基づく）
- タグフィルタ：学年帯（小/中）、難易度（基礎/標準/発展）
- 結果表示：得点、教科別内訳、難易度別内訳、解説（番号クリックで表示）
- 可視化：5教科レーダー、履歴（最近10回折れ線）
- 端末内履歴（localStorage）：ログイン不要で同一端末内に蓄積
- 合言葉（パスフレーズ）による簡易アクセス制御

## 6. 技術構成（要点）
- フロント：HTML / CSS / Vanilla JS
- 配信：GitHub Pages（静的ホスティング）
- 履歴：localStorage（サーバ不要）
- PWA：Service Worker（オフライン・キャッシュ）

## 7. 工夫した点（就活で話すポイント）
- **品質戦略**：理科・社会は誤情報リスクを避けるため固定問題中心に増量（生成依存を避ける）
- **出題の偏り対策**：25問内重複禁止＋pattern偏り抑制＋難易度比率（3:4:3）を維持
- **分析設計**：正誤×時間×再訪（visits）で、行動傾向を文章化し次の改善行動に繋げる
- **公開運用**：Service Workerのキャッシュ問題を前提に、更新反映手順（強制リロード／CACHE_NAME更新）を整備

## 8. 今後の改善
- 理科・社会の固定問題を入試レベルで100〜300問へ拡充（品質担保のまま密度増）
- 数学：図形・証明穴埋め・関数解釈の難問比率を増やし、選択肢の誤答設計を強化
- 任意：サーバ同期（ログイン）で複数端末の履歴統合（将来案）
