# 要件定義（Requirements）

## スコープ
- 対象：小学校〜中学校の学習範囲
- 教科：国語 / 数学 / 英語 / 理科 / 社会（5教科）

## 機能要件
### 出題
- 25問を提示（各教科5問）
- 出題順はランダム
- 25問内で同一問題が重複しない
- 似た出題パターンの連発を抑制する（patternによる制御）
- 出題条件フィルタ：
  - 学年帯タグ：小/中
  - 難易度タグ：基礎/標準/発展
- 難易度比率：基礎20% / 標準50% / 発展30% をなるべく維持

### 解答・採点
- 選択肢クリックで回答
- 「採点」押下で結果表示
- 途中で前後移動可能
- 解答リセット可能

### 結果表示
- 総合得点（正答数/25、正答率）
- 教科別内訳（正答数、正答率、平均時間）
- 難易度別内訳
- レーダーチャート（5教科正答率）
- 問題番号クリックで解説表示

### 分析（文章）
- 正誤×解答時間の傾向から短い所見を生成
- 弱点が明確でない（偏りが少ない）場合は「弱点なし」と明記する

### 履歴（この端末）
- 採点結果を端末内に蓄積（ログイン不要）
- 履歴パネルの開閉ボタン
- 最近10回の正答率推移を折れ線で表示
- 全期間の平均（全体・教科別）を表示
- 履歴の全削除

### アクセス制御
- 合言葉（パスフレーズ）で開始可
- 合言葉未入力の場合は開始不可

## 非機能要件
- GitHub Pagesで動作する（サーバ不要）
- モバイル/PC両対応（レスポンシブ）
- キャッシュの影響（SW）で更新が反映されない場合に対処手順を用意

## 制約
- 端末内履歴のみ（複数端末間で同期しない）
- 社会/理科は誤情報リスク回避のため「固定問題中心」で増量する
